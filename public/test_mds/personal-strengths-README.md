# Personal Strengths Test - Technical Documentation

## 1. Test Overview

**Test Name:** Personal Strengths Assessment (Character Strengths)  
**Test Code:** `personal-strengths`  
**Test ID:** `test_010`  
**Purpose:** Identify top 5 personal character strengths that shape thinking, behavior, and life approach  
**Measures:** 24 character strengths across 6 virtue categories  
**Questions:** 100 statements  
**Duration:** 10-12 minutes  
**Response Type:** 5-point Likert scale (Inaccurate → Accurate)  
**Weight in Analysis:** 15%

---

## 2. Test Description

Everyone has a unique set of personal strengths—qualities that shape how you think, feel, and move through the world. This personality test helps you uncover your strongest traits, from creativity and leadership to compassion, discipline, and everything in between.

**What It Measures:**

Personal strengths are character traits that:
- Come naturally to you
- Energize rather than drain you
- Feel authentic when expressed
- Help you perform at your best
- Shape your unique contribution to the world

**Based on VIA Character Strengths:**

The test is based on the VIA (Values in Action) Classification of Character Strengths, developed by psychologists Christopher Peterson and Martin Seligman. This framework identifies 24 universal character strengths organized into 6 core virtues.

**Why It Matters:**

Understanding your strengths helps you:
- Appreciate yourself and what you offer the world
- Make aligned decisions about career and life
- Choose situations that leverage natural abilities
- Increase self-awareness and personal growth
- Build on what already comes naturally
- Create environments where you thrive

---

## 3. Test Format

**Structure:**
- 100 self-descriptive statements
- 5-point Likert scale for each statement
- No time limit (typically 10-12 minutes)
- Statements distributed across 24 strengths

**Scale:**
1. **Inaccurate** - This statement does not describe me at all
2. **Mostly Inaccurate** - This statement rarely describes me
3. **Neutral** - Uncertain or sometimes true
4. **Mostly Accurate** - This statement often describes me
5. **Accurate** - This statement describes me very well

**Question Distribution:**
- Each strength: ~4 questions
- 24 strengths total
- Questions intermixed (not grouped by strength)

## 7. Scoring System

### Raw Score Calculation

```python
def calculate_raw_scores(answers):
    """
    Calculate raw scores for each character strength
    """
    # Initialize scores for all 24+ strengths
    scores = {
        "creativity": 0,
        "curiosity": 0,
        "judgment": 0,
        "love_of_learning": 0,
        "perspective": 0,
        "bravery": 0,
        "perseverance": 0,
        "honesty": 0,
        "zest": 0,
        "love": 0,
        "kindness": 0,
        "social_intelligence": 0,
        "teamwork": 0,
        "fairness": 0,
        "leadership": 0,
        "forgiveness": 0,
        "humility": 0,
        "prudence": 0,
        "self_regulation": 0,
        "awe": 0,
        "gratitude": 0,
        "optimism": 0,
        "humor": 0,
        "spirituality": 0,
        "discipline": 0,
        "stress_management": 0,
        "tradition": 0,
        "sociability": 0,
        "independence": 0,
        "integrity": 0
    }
    
    # Question mapping (strength → question IDs)
    question_map = {
        "creativity": [17, 42, 66, 92],
        "curiosity": [8, 33, 58, 83],
        "judgment": [20, 45, 70, 95],
        "love_of_learning": [25, 50, 75, 100],
        "perspective": [2, 27, 52, 77],
        # ... continue for all strengths
    }
    
    # Reverse scored questions
    reverse_scored = [21, 38, 62, 87, 91, ...]
    
    # Calculate scores
    for answer in answers:
        question_id = answer['question_id']
        response = answer['response']  # 1-5
        
        # Handle reverse scoring
        if question_id in reverse_scored:
            response = 6 - response
        
        # Add to appropriate strength(s)
        for strength, questions in question_map.items():
            if question_id in questions:
                scores[strength] += response
    
    return scores
```

### Normalization to Percentage

```python
def normalize_scores(raw_scores):
    """
    Normalize raw scores to 0-100 percentage
    """
    normalized = {}
    
    for strength, raw_score in raw_scores.items():
        # Each strength has ~4 questions
        # Min = 4, Max = 20
        num_questions = 4
        min_possible = num_questions * 1
        max_possible = num_questions * 5
        
        # Normalize to 0-100
        percentage = (
            (raw_score - min_possible) / (max_possible - min_possible)
        ) * 100
        
        normalized[strength] = percentage
    
    return normalized
```

### Determine Top 5 Strengths

```python
def get_top_strengths(normalized_scores, count=5):
    """
    Get top N strengths sorted by score
    """
    # Sort by score descending
    sorted_strengths = sorted(
        normalized_scores.items(),
        key=lambda x: x[1],
        reverse=True
    )
    
    # Take top N
    top_strengths = sorted_strengths[:count]
    
    return [
        {
            "strength": strength,
            "score": score,
            "rank": idx + 1
        }
        for idx, (strength, score) in enumerate(top_strengths)
    ]
```

## 10. Database Schema

### strengths_questions

```sql
CREATE TABLE strengths_questions (
    id SERIAL PRIMARY KEY,
    question_number INTEGER NOT NULL,
    text TEXT NOT NULL,
    strengths TEXT[] NOT NULL,  -- Array of strength names this question measures
    reverse_scored BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMP DEFAULT NOW()
);

-- Index for fast lookups
CREATE INDEX idx_strengths_questions_strengths ON strengths_questions USING GIN(strengths);
```

### strengths_results

```sql
CREATE TABLE strengths_results (
    id SERIAL PRIMARY KEY,
    user_id UUID REFERENCES users(id),
    test_session_id UUID,
    
    -- All scores (JSONB for flexibility with 24+ strengths)
    all_scores JSONB NOT NULL,
    
    -- Top 5 strengths
    top_5_strengths JSONB NOT NULL,
    
    -- Virtue category averages
    virtue_breakdown JSONB NOT NULL,
    
    completed_at TIMESTAMP DEFAULT NOW(),
    duration_seconds INTEGER
);

-- Indexes
CREATE INDEX idx_strengths_results_user ON strengths_results(user_id);
CREATE INDEX idx_strengths_results_top ON strengths_results USING GIN(top_5_strengths);
```

### strengths_answers

```sql
CREATE TABLE strengths_answers (
    id SERIAL PRIMARY KEY,
    result_id INTEGER REFERENCES strengths_results(id),
    question_number INTEGER NOT NULL,
    response INTEGER NOT NULL,  -- 1-5
    answered_at TIMESTAMP DEFAULT NOW()
);

-- Index for fast lookups
CREATE INDEX idx_strengths_answers_result ON strengths_answers(result_id);
```

## 11. API Endpoints

### GET /api/tests/strengths/questions
Returns all 100 questions

**Response:**
```json
{
  "success": true,
  "data": [
    {
      "id": 1,
      "text": "I am welcoming to friends and acquaintances",
      "scale": {
        "min": 1,
        "max": 5,
        "labels": {
          "1": "Strongly Disagree",
          "5": "Strongly Agree"
        }
      }
    }
  ]
}
```

### POST /api/tests/strengths/submit
Accepts answers and returns calculated result

**Request:**
```json
{
  "user_id": "uuid",
  "answers": [
    {"question_number": 1, "response": 5},
    {"question_number": 2, "response": 4},
    ...
  ],
  "duration_seconds": 660
}
```

**Response:**
```json
{
  "success": true,
  "result": {
    "top_5_strengths": [
      {
        "rank": 1,
        "strength": "discipline",
        "name": "Discipline",
        "score": 50
      }
    ],
    "all_scores": {...},
    "virtue_breakdown": {...}
  }
}
```

### GET /api/users/{user_id}/tests/strengths/result
Returns user's saved result
